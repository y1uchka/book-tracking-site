Курсовой проект: «Трекер чтения книг»

1. Цель проекта
Система должна помогать пользователям:
	•	планировать чтение;
	•	отслеживать прогресс;
	•	анализировать статистику;
	•	получать мотивационные стимулы.

2. Функциональные требования

Основные функции:
	/////•	Добавление книг 
	/////•	ввод названия, автора, общего количества страниц;
	/////•	сохранение данных в структурированном виде (массив объектов).
	/////•	Отметка прогресса
	/////•	указание прочитанных страниц/глав;
	/////•	обновление статуса книги («читаю», «закончил», «отложил»).
	•	Постановка целей
	•	установка дневной/недельной нормы (например, 50 стр./день);
	•	контроль выполнения целей с визуальной индикацией.
	•	Мотивационные механизмы
	•	автоматические сообщения при достижении целей;
	•	награды за серии дней чтения;
	•	цитаты или факты о пользе чтения.
	•	Статистика
	//////•	количество завершённых книг;
	•	средняя скорость чтения (стр./день);
	•	график прогресса за период (неделя/месяц).

Технические требования:
	/////•	использование массивов для хранения списка книг;
	/////•	применение объектов для описания каждой книги (свойства: title, author, totalPages, readPages, status, goal, startDate и т. д.);
	/////•	работа со строками для валидации ввода и формирования сообщений;
	////•	взаимодействие с пользователем через HTML/CSS-интерфейс (форма ввода, таблицы, графики);
	•	сохранение данных в localStorage для офлайн-доступа.

3. Архитектура проекта

Структура файлов:
	•	index.html — разметка интерфейса;
	•	style.css — стилизация;
	•	app.js — основной скрипт (разбит на модули: работа с данными, UI, логика целей и статистики);
	•	data.js — хранилище книг (массив объектов).

Ключевые компоненты JS:
	•	Массивы: books — список всех книг; goals — массив целей.
	•	Объекты: каждый элемент массива books является объектом.
	•	Строки: валидация ввода (проверка на пустоту, спецсимволы), формирование уведомлений.
	•	Работа с датами: расчёт длительности чтения, определение дедлайнов целей.

4. Интерфейс пользователя

Элементы HTML:
	•	форма добавления книги (поля: название, автор, страницы);
	•	таблица списка книг с колонками: название, прогресс (в %), статус, цель;
	•	блок целей с индикатором выполнения;
	•	секция статистики с графиками (используем Canvas или библиотеку Chart.js);
	•	модальное окно для отметки прочитанных страниц.
    
Взаимодействие:
	•	кнопки действия («Добавить книгу», «Отметить прогресс», «Посмотреть статистику»);
	•	всплывающие уведомления при достижении целей;
	•	фильтрация книг по статусу («все», «читаю», «закончил»).
